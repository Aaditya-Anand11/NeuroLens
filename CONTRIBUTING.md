## How to Contribute

### Reporting Bugs

1. Check the [Issues](https://github.com/Aaditya-Anand11/NeuroLens/issues) page first.
2. Open a new issue with:
   - A clear title
   - Steps to reproduce
   - Expected vs. actual behavior
   - Your OS, Python version, and Node.js version

### Suggesting Features

Open an issue with the `enhancement` label. Describe:
- The problem you're solving
- Your proposed solution
- Any alternatives you considered

### Submitting Pull Requests

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** following the code style below
4. **Test** your changes locally
5. **Commit** with a clear message: `feat: add feature description`
6. **Push** and open a Pull Request against `main`

### PR Requirements

- All PRs must include a description of what changed and why
- Backend changes require updated docstrings
- New features should include relevant tests
- Keep changes focused — one feature or fix per PR

## Code Style

**Backend (Python):**
- Follow PEP 8
- Type hints on all public functions
- Docstrings on all modules and public classes

**Frontend (TypeScript/React):**
- Follow the existing component patterns
- Use TypeScript types, avoid `any`
- Keep components focused and small
